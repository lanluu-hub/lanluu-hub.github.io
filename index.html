Connect Four AI

Detail

This project develops and evaluates AI agents for Connect Four using both reinforcement learning and classical search-based methods. The primary agent is a Deep Q-Network (DQN) implemented in PyTorch and trained in a Gymnasium environment, where it learns optimal strategies through experience replay and temporal difference learning. The network uses a 3-layer architecture with a 42-dimensional state representation of the 6×7 board and separate policy and target networks to stabilize training. Training includes epsilon-greedy exploration, controlled epsilon decay, and evaluation metrics such as win rates and loss trends.

In parallel, a Minimax agent with alpha-beta pruning is implemented as a strong rule-based baseline. It uses a heuristic evaluation function that considers center control, consecutive piece patterns, and opponent threats, along with action ordering to improve search efficiency. The project also explores self-play training, allowing DQN agents to progressively improve by competing against earlier versions of themselves instead of static opponents.

My contribution

I implemented the Minimax agent with alpha-beta pruning and designed its heuristic evaluation function based on board control, sequence detection, and threat blocking. I created the DQNPlayer wrapper to load trained models and integrate them into the evaluation pipeline for fair comparisons. I also built the self-play training framework that iteratively trains new agents against previous generations and visualizes win-rate progression, and ensured compatibility and correctness of older model evaluations within the testing system.

Strings DSL Interpreter

Detail

The CS358 Interpreter is a complete expression language implementation built in Python that supports core functional programming constructs and an extensible string DSL. The system consists of three main components: an abstract syntax tree (AST) definition and evaluator in interp.py, a Lark-based parser with concrete syntax grammar in expr.lark, and an AST transformer in parse_run.py. The core language includes arithmetic and boolean operations with short-circuit evaluation, conditional expressions, variable binding with lexical scoping via immutable environments, and first-class functions with closure support enabling recursion. The interpreter uses a tree-walking evaluation pattern with environment passing, models mutable state through memory locations (singleton lists), and includes comprehensive type checking with runtime error reporting. The string DSL extension adds concatenation (++), replace (with first-match semantics), reverse, uppercase, and lowercase operations. The grammar uses Earley parsing with explicit precedence rules, handles escape sequences in string literals via ast.literal_eval, and treats true/false as parsed identifiers converted to boolean literals during AST transformation.

My contribution

I implemented the complete AST node definitions and the recursive evaluator with support for all core language features including arithmetic, boolean logic, variable binding, functions with closures, and conditionals. I designed and implemented the string DSL extension with Concat, Replace, Reverse, Uppercase, and Lowercase operations. I built the Lark grammar (expr.lark) with proper operator precedence and rules for parsing all language constructs, and implemented the transformer in parse_run.py to convert parse trees to ASTs. I also added mutable variable support through memory locations, imperative operations (sequencing, assignment, show/read), and comprehensive error handling with type validation.
